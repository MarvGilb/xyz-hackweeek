XYZ Splunk Viewer

Idea

  • To build a generic application which can be used for a continuous visualization of Splunk logs which are containing geo-related data.

What do you see?

  • We have chosen the xyz-osm-base logs as an example dataset. Every point on the map displays accessed xyz-osm-base tiles. So you are able to get a feeling which regions were the hotspots of our hackweek. Due to the reason that we have a front-cache in place you can only see requested tiles from cache-misses.

Controls?

  • Log: Choose Log-Source (placeholder)
  • Day: Choose the day from which you want to see the log-data
  • Hour: Choose the hour from which you want to see the log-data
  • AutoPlay: If it is activated, the time-slider iterates (3s interval) through the chosen day timeframe automatically
  • HERE-Offices: Enable/Disable the visualization of the biggest HERE-Offices

Components

  • Data-Loader (Node-Script running as Lambda):
    The data-loader is running as a Lambda function in AWS. It is frequently requesting log-data from our HERE-Splunk Server. It processes pre-defined Splunk-Searches which have to return geo-related data. For each Splunk-Search is a regular expression defined which parses the geolocation and some attributes of interest. Finally those pre-processed datasets get uploaded into a XYZ-Space.
  • XYZ-Splunk Viewer:
    The UI-Component of the project. It uses Tangram and Leaflet to visualize the xyz-spaces which provides the data.

Used Technologies

  • HERE-Splunk-Server, AWS-Lambda, AWS-S3, XYZ-Spaces, Tangram, Leaflet, Nodejs, Javascript
